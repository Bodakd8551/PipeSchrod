01Introduction

The Schrödinger equation, written in 1926, encodes every observable property of a quantum system. Yet for most physically interesting potentials — the Cornell funnel that confines quarks, the Woods-Saxon well that shapes atomic nuclei — it yields no analytic solution. Numerical matrix methods are indispensable.

This article presents four progressively powerful matrix methods for the radial Schrödinger equation, each implemented in PipeSchrod — a Python package built around the >> pipe operator. Every calculation in the article was produced by a single chain of >> steps: 

from pipeschrod import PipeSchrod
from pipeschrod.steps import Cornell, Grid, Solve, Compare, Plot, Export

(PipeSchrod(m1=1.4495, m2=1.4495)
 >> Cornell(alpha=0.5317, b=0.1497)
 >> Grid(N=200, rmax=20.0)
 >> Solve("all")
 >> Compare()
 >> Plot("dashboard")
 >> Export("csv", path="charmonium.csv"))

The benchmark system throughout is charmonium — the cc̄ meson family. The J/ψ was discovered in 1974 (the “November Revolution” of particle physics), its spectrum is precisely known from experiment, and its canonical Cornell potential parameters give a compact, well-defined numerical problem. Parameters used: α = 0.5317, b = 0.1497 GeV², m₁ = m₂ = 1.4495 GeV.

02The Radial Schrödinger Equation

For two particles in a spherically symmetric potential V(r), writing ψ = [u(r)/r]·Yₗᵐ separates the angular parts exactly, leaving a one-dimensional eigenvalue problem for u(r):

-ℏ² d²u ℏ²L(L+1) ---- ---- + --------- u(r) + V(r) u(r) = E · u(r) 2μ dr² 2μr²
Eq.(1) — Radial Schrödinger equation. μ = m₁m₂/(m₁+m₂) is the reduced mass.

Boundary conditions u(0)=0 and u(∞)=0 quantise the energy into a discrete spectrum E₁ < E₂ < E₃… The term L(L+1)/r² is the centrifugal barrier: L=0 gives S-states, L=1 gives P-states, and so on. All four PipeSchrod methods solve exactly this equation, differing only in how they represent the kinetic energy operator d²/dr² as a matrix.

The Cornell Potential

The Cornell potential combines the one-gluon-exchange Coulomb interaction (perturbative QCD at short range) with a linearly rising confinement term (the colour flux tube at long range):

V(r) = -4α/(3r) + b·r
Eq.(2) — Cornell potential. The 4/3 is the SU(3) colour factor for a quark–antiquark singlet.

Figure 1. The Cornell potential V(r) with α=0.5317, b=0.1497 GeV², and the first four S-state energy levels shown as dashed lines. The competing Coulomb (short range) and linear confinement (long range) terms create the characteristic funnel shape.

Discretisation

All four methods replace the continuous r with a uniform grid of N=200 points from r₀=0.1 to rₘₐₓ=20.0 GeV⁻¹ (step h=0.1 GeV⁻¹). The wave function u(r) becomes a column vector, and H·u=E·u becomes a standard matrix eigenvalue problem solved by scipy.linalg.eigh. Every eigenvalue is a total meson mass in GeV; the binding energy is B.E. = E − (m₁+m₂).

03Introducing PipeSchrod

Part of the Pipe* family

PipeFrame · PipePlotly · PipeScrape · PipeSchrod

PipeSchrod extends the Pipe* ecosystem to quantum physics. Every step of a calculation — defining the system, choosing a potential, setting the grid, running solvers, plotting, exporting — is a >> step on a PipeSchrod object.

The >> operator is immutable: each step returns a new object, leaving the original unchanged. This means you can define a base pipe once and branch it into multiple solver chains without recomputing the potential or grid:

# Define once, branch freely
base = (PipeSchrod(m1=1.4495, m2=1.4495)
        >> Cornell(alpha=0.5317, b=0.1497)
        >> Grid(N=200, rmax=20.0))

fgh_pipe  = base >> Solve("FGH")       # base unchanged
sal_pipe  = base >> Solve("Salpeter")   # independent branch
all_pipe  = base >> Solve("all")        # all four solvers

Figure 2. The PipeSchrod >> chain: each box is a step class; arrows show the immutable data flow. Every >> returns a new object, so any sub-chain can be stored, reused, or extended independently.

Available Steps

Potential steps — Cornell, Harmonic, Coulomb, WoodsSaxon, Morse. Grid — sets N, rmax, r₀. Solve — runs any of "Matrix", "Numerov", "FGH", "Salpeter", or "all". Compare — rich terminal comparison table. Summary — eigenvalue table with B.E., nodes, ⟨r⟩, rms. Plot — nine figure types (dashboard, wavefunctions, densities, potential, spectrum, compare_wf, compare_E, convergence, sensitivity). Export — saves CSV, JSON, or multi-method comparison CSV.

04Method 1 — Standard Matrix Method

O(h²) · Tridiagonal · Fastest Build

Second-Order Finite Difference

The most direct approach: replace d²u/dr² with the classic three-point central difference and assemble a tridiagonal Hamiltonian.

The second derivative is approximated by the lowest-order centred difference, exact for polynomials up to degree 3 and with truncation error O(h²):

u"(rᵢ) ≈ (uᵢ₋₁ - 2uᵢ + uᵢ₊₁) / h² + O(h²)
Eq.(3) — Second-order three-point finite difference.

Adding the rest masses (m₁+m₂) so eigenvalues equal total meson masses, and the centrifugal term V𝒜(rᵢ)=V(rᵢ)+L(L+1)/(2μrᵢ²), the Hamiltonian matrix is perfectly tridiagonal:

Hᵢᵢ = (m₁+m₂) + 1/(μh²) + V𝒜(rᵢ) diagonal Hᵢ,ᵢ±₁ = -1/(2μh²) off-diagonal
Eq.(4) — Matrix method Hamiltonian. Tridiagonal structure → O(N²) storage. 
# PipeSchrod usage
pipe = (PipeSchrod(m1=1.4495, m2=1.4495)
        >> Cornell(alpha=0.5317, b=0.1497)
        >> Grid(N=200, rmax=20.0)
        >> Solve("Matrix")
        >> Summary())

# Equivalent explicit implementation
diag = (m1+m2) + 1.0/(mu*h**2) + Veff
off  = np.full(N-1, -0.5/(mu*h**2))
H    = np.diag(diag) + np.diag(off,1) + np.diag(off,-1)
E, psi = eigh(H)     # all N eigenvalues at once
Result: E₁ = 3.04248 GeV  |  B.E. = 143.5 MeV  |  CPU ≈ 8 ms  |  9 bound states found.

05Method 2 — Matrix Numerov

O(h⁴) · Dense B⁻¹A · 4th-Order

Fourth-Order Numerov B⁻¹A Formulation

Achieves fourth-order accuracy through the Numerov B⁻¹A matrix product — the standard algorithm in nuclear structure codes, derived in the author’s doctoral research.

Adding the forward and backward Taylor expansions of u(rₙ±h) through h⁴ and using u″=f(r)u to eliminate the u⁴″ term yields Numerov’s fourth-order recurrence:

uₙ₊₁ = [(2 - 5h²fₙ/6)uₙ - (1 + h²fₙ₋₁/12)uₙ₋₁] / (1 + h²fₙ₊₁/12)
Eq.(5) — Numerov recurrence. Error O(h⁶) per step → O(h⁴) overall convergence.

Casting this as a matrix eigenvalue problem using tridiagonal matrices A and B:

A = tridiag(1,-2,1)/h² B = tridiag(1,10,1)/12 T = -(1/2μ) · B⁻¹ · A ← 4th-order kinetic (dense matrix) H = (m₁+m₂)·I + T + diag(V𝒜)
Eq.(6) — Numerov B⁻¹A formulation from the author’s doctoral thesis. 
pipe = (base >> Solve("Numerov") >> Summary())

# Core implementation
A = (np.diag(e1,-1)-2*np.diag(e0)+np.diag(e1,1))/h**2
B = (np.diag(e1,-1)+10*np.diag(e0)+np.diag(e1,1))/12.0
T = -(1/(2*mu)) * np.linalg.inv(B) @ A   # dense B⁻¹A product
np.fill_diagonal(T, T.diagonal()+(m1+m2))
E, psi = eigh(T + np.diag(Veff))
Result: E₁ = 3.04306 GeV  |  B.E. = 144.1 MeV  |  CPU ≈ 202 ms  |  0.6 MeV more accurate than Matrix at same N.

06Method 3 — Fourier Grid Hamiltonian

O(h⁶) · 5-Point Stencil · Fastest Convergence

Sixth-Order Spectral-Class Accuracy

Extends the stencil to five points, eliminating both the h² and h⁴ error terms simultaneously. The fastest-converging non-relativistic method and the preferred solver for parameter scans.

The Fourier Grid Hamiltonian method (Colbert & Miller, 1992) is grounded in the insight that kinetic energy is diagonal in momentum space. For potentials with a Coulomb singularity, the equivalent sixth-order five-point stencil achieves the same spectral-class accuracy while remaining numerically stable:

u"(rᵢ) ≈ [-uᵢ₋₂/12 + 4uᵢ₋₁/3 - 5uᵢ/2 + 4uᵢ₊₁/3 - uᵢ₊₂/12] / h²
Eq.(7) — Five-point 6th-order stencil. Eliminates h² and h⁴ truncation simultaneously.

This gives a pentadiagonal Hamiltonian with five non-zero diagonals:

Hᵢᵢ = (m₁+m₂) + (5/2)/(2μh²) + V𝒜(rᵢ) Hᵢ,ᵢ±₁ = -(4/3)/(2μh²) Hᵢ,ᵢ±₂ = +(1/12)/(2μh²)
Eq.(8) — FGH pentadiagonal matrix elements. Two extra diagonals carry the 6th-order accuracy.

Figure 3. Stencil weight comparison (scaled for display). The Matrix method uses three points with coefficients [1,−2,1]/h². The Numerov B⁻¹A achieves 4th-order via weighted three-point coupling. The FGH uses five points with [−1/12, 4/3, −5/2, 4/3, −1/12]/h², eliminating both h² and h⁴ error terms.

pipe = (base >> Solve("FGH") >> Summary())

# Core implementation
fac = 1.0/(2.0*mu*h**2)
d0,d1,d2 = fac*2.5, -fac*4.0/3.0, fac/12.0
diag = (m1+m2) + d0 + Veff
H = (np.diag(diag)
     + np.diag(np.full(N-1,d1),1)  + np.diag(np.full(N-1,d1),-1)
     + np.diag(np.full(N-2,d2),2)  + np.diag(np.full(N-2,d2),-2))
E, psi = eigh(H)
Result: E₁ = 3.04911 GeV  |  B.E. = 150.1 MeV  |  CPU ≈ 71 ms  |  Sub-MeV convergence at N≈75 vs N≈200 for Matrix.

07Method 4 — Spinless Salpeter Equation

Relativistic · T=√(m²+p²) · Semi-Covariant

Full Relativistic Kinematics

Replaces p²/2μ with the exact relativistic dispersion √(m²+p²). Essential for charmonium and bottomonium where quark velocity v/c ≈ 0.3–0.5.

The three methods above use the non-relativistic kinetic energy T=p²/2μ. For charmonium, where the charm quark velocity v/c≈0.3–0.5, this introduces systematic errors of 5–10% in the kinetic energy. The spinless Salpeter equation (SSE) corrects this by using the exact relativistic dispersion for each quark:

[√(m₁²+p²) + √(m₂²+p²) + V(r)] ψ(r) = E · ψ(r)
Eq.(9) — Spinless Salpeter Equation. Each √ term is the exact relativistic kinetic energy of one quark.

The square-root operator √(m²+p²) is non-local in position space. PipeSchrod handles this via the Numerov eigenspace method: diagonalise the Numerov B⁻¹A matrix to obtain discrete momentum modes kₙ², apply the relativistic dispersion T(kₙ)=√(m₁²+kₙ²)+√(m₂²+kₙ²) to each mode, then reconstruct T𝕊ₐₗ in position space:

Step 1: diagonalise Numerov: T𝑁𝑅 vₙ = (kₙ²/2μ) vₙ Step 2: T(kₙ) = √(m₁²+kₙ²) + √(m₂²+kₙ²) Step 3: T𝕊ₐₗ = V𝖊 · diag[T(kₙ)] · V𝖊ᵀ Step 4: solve [T𝕊ₐₗ + diag(V𝒜)] u = E u
Eq.(10) — Numerov eigenspace method for the Salpeter relativistic kinetic energy. 
pipe = (base >> Solve("Salpeter") >> Summary())

# Core: momentum eigenspace method
T_NR = -(1/(2*mu)) * np.linalg.inv(B) @ A    # Numerov kinetic
k2, Vk = eigh(T_NR * 2*mu)                   # momentum modes k²
k2     = np.clip(k2, 0, None)
Tk     = np.sqrt(m1**2+k2) + np.sqrt(m2**2+k2)  # relativistic T(k)
T_sal  = Vk @ np.diag(Tk) @ Vk.T              # back to position space
E, psi = eigh(T_sal + np.diag(Veff))
Result: E₁ = 2.96917 GeV  |  B.E. = 70.2 MeV  |  CPU ≈ 291 ms  |  Relativistic shift = −73.3 MeV vs Matrix → closer to experimental J/ψ = 3.097 GeV.

09The PipeSchrod API in Full

Complete Chain Patterns

from pipeschrod import PipeSchrod
from pipeschrod.steps import Cornell, Grid, Solve, Compare, Summary, Plot, Export

# ── Pattern 1: single solver with full output ─────────────────────
(PipeSchrod(m1=1.4495, m2=1.4495)
 >> Cornell(alpha=0.5317, b=0.1497)
 >> Grid(N=200, rmax=20.0)
 >> Solve("FGH")
 >> Summary(n_states=6)
 >> Plot("dashboard", save="./figures")
 >> Export("csv", path="fgh_result.csv"))

# ── Pattern 2: all four solvers, compare and export ───────────────
(PipeSchrod(m1=1.4495, m2=1.4495)
 >> Cornell(alpha=0.5317, b=0.1497)
 >> Grid(N=200, rmax=20.0)
 >> Solve("all")
 >> Compare(n_states=6)
 >> Plot("compare_E", "compare_wf", save="./figures")
 >> Export("csv_compare", path="comparison.csv"))

# ── Pattern 3: immutable branching ───────────────────────────────
base = (PipeSchrod(m1=1.4495, m2=1.4495)
        >> Cornell(alpha=0.5317, b=0.1497)
        >> Grid(N=200, rmax=20.0))

fgh = base >> Solve("FGH")      # base unchanged
sal = base >> Solve("Salpeter")  # independent branch

print(fgh.result.bound_energies[0])   # 3.04911 GeV
print(sal.result.bound_energies[0])   # 2.96917 GeV
delta = (fgh.result.bound_energies[0] - sal.result.bound_energies[0]) * 1000
print(f"Relativistic shift: {delta:.1f} MeV")  # 73.3 MeV

# ── Pattern 4: P-wave states (L=1) ───────────────────────────────
(PipeSchrod(m1=1.4495, m2=1.4495, L=1, S=1, J=1)
 >> Cornell(alpha=0.5317, b=0.1497)
 >> Grid(N=200, rmax=20.0)
 >> Solve("FGH")
 >> Summary())
# 1P: 3.500 GeV  (exp. chi_c(1P) centroid: 3.511 GeV)

# ── Pattern 5: bottomonium ────────────────────────────────────────
(PipeSchrod(m1=4.67, m2=4.67)
 >> Cornell(alpha=0.471, b=0.192)
 >> Grid(N=200, rmax=20.0)
 >> Solve("FGH", "Salpeter")
 >> Compare())
# FGH: 9.084 GeV | Salpeter: 9.006 GeV | exp. Upsilon(1S): 9.460 GeV

# ── Pattern 6: Cornell with hyperfine (type 2) ───────────────────
(PipeSchrod(m1=1.4495, m2=1.4495)
 >> Cornell(alpha=0.5317, b=0.1497, pot_type=2)
 >> Grid(N=200, rmax=20.0)
 >> Solve("Matrix", "FGH")
 >> Compare())
# Hyperfine shifts: +24.6 MeV (triplet), -73.9 MeV (singlet)

10Conclusion

Four matrix methods for the radial Schrödinger equation — each a natural progression from the last — have been presented, derived from first principles, and demonstrated on the charmonium system using the pipe-chained PipeSchrod package.

The progression from Matrix to Salpeter is one of both numerical efficiency and physical completeness. Moving from second-order to sixth-order finite differences (Matrix → FGH) reduces the required grid size by a factor of three with no change in the physics. Moving from non-relativistic to relativistic kinematics (FGH → Salpeter) shifts every eigenvalue by 70–190 MeV — a genuinely different physical prediction that brings the 1S charmonium mass from 3.049 GeV to 2.969 GeV, significantly closer to the experimental J/ψ mass of 3.097 GeV.

The >> syntax of PipeSchrod makes the connection between code and physics explicit: each step in the chain corresponds directly to one conceptual step in the calculation. Branching the base pipe into multiple solver chains is as natural as it is immutable — no state is ever accidentally shared or overwritten.

The same framework extends without modification to nuclear shell models (Woods-Saxon potential), molecular vibrations (Morse potential), hydrogen-like atoms (Coulomb potential), and any other spherically symmetric quantum system. Change two lines of code — the potential step and the quark masses — and PipeSchrod computes a new spectrum.
